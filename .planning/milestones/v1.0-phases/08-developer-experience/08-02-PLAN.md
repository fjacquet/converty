<task type="auto">
  <name>Task 1: Install and configure Husky + lint-staged</name>
  <files>
    package.json

    .gitignore
  </files>
  <action>
**1. Install dependencies:**

```bash
npm install --save-dev husky@^9.1.7 lint-staged@^16.2.7
```

**2. Initialize Husky (modern v9 API):**

```bash
npx husky init
```

This automatically:

- Creates `.husky/` directory
- Creates `.husky/pre-commit` file
- Adds `"prepare": "husky"` to package.json scripts

**3. Configure lint-staged:**

Create `.lintstagedrc.json` in project root:

```json
{
  "*.{ts,tsx,js,mjs,cjs}": [
    "biome check --write --files-ignore-unknown=true --no-errors-on-unmatched"
  ]
}
```

**Flags explained:**

- `--write`: Auto-fix lint and format issues
- `--files-ignore-unknown=true`: Don't error on non-JS/TS files
- `--no-errors-on-unmatched`: Don't fail if no files match pattern (e.g., committing only images)

**4. Update .husky/pre-commit hook:**

Replace content of `.husky/pre-commit` with:

```bash
#!/bin/sh
npx lint-staged
```

Make sure the file is executable:

```bash
chmod +x .husky/pre-commit
```

**5. Update .gitignore:**

Add Husky internal files to `.gitignore` (only commit hook scripts, not internals):

```
# Husky
.husky/_
```

**Important notes:**

- Modern Husky v9 uses `husky init` (not deprecated `husky install`)
- The `prepare` script runs automatically on `npm install`
- Only `.husky/pre-commit` and other hook scripts are version-controlled
- `.husky/_/` internal directory is not committed (added to .gitignore)
