Create IP classification calculation logic in `ip-classifier.ts`:

1. **Define IPClassification interface:**

```typescript
export interface IPClassification {
  ipClass: "A" | "B" | "C" | "D" | "E" | null; // null for IPv6 (no classes)
  isPrivate: boolean;
  isPublic: boolean;
  isSpecial: boolean; // loopback, link-local, etc.
  rangeType: string; // From ipaddr.js range()
  rangeDescription: string; // Human-readable description
  ipVersion: 4 | 6;
  normalizedIP: string; // Normalized form of input
}
```

1. **Create classifyIPAddress function:**

- Use `ipaddr.isValid()` for validation
- Use `ipaddr.parse()` for parsing
- Use `addr.range()` for range detection (returns "private", "unicast", "loopback", "linkLocal", "uniqueLocal", etc.)
- For IPv4, determine class from first octet:
  - 0: "special" (not Class A)
  - 1-126: Class A
  - 127: "special" (loopback, not Class A)
  - 128-191: Class B
  - 192-223: Class C
  - 224-239: Class D (multicast)
  - 240-255: Class E (reserved)
- For IPv6: ipClass = null (IPv6 has no classes)
- Map rangeType to human-readable descriptions:
  - "private" -> "Private network (RFC 1918)"
  - "unicast" -> "Public/Routable address"
  - "loopback" -> "Loopback address"
  - "linkLocal" -> "Link-local address"
  - "uniqueLocal" -> "Unique local address (IPv6 private)"
  - etc.

1. **Export from types.ts:**
Add `export type { IPClassification } from "./ip-classifier";` to types.ts

Key implementation notes:

- Import ipaddr from "ipaddr.js"
- Throw error for invalid IPs (store catches and sets error state)
- Handle edge cases: 0.0.0.0 (unspecified), 255.255.255.255 (broadcast)
