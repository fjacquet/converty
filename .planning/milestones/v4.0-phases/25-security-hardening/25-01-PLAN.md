  <name>Task 1: Refactor getUrlParams to use Map-based storage</name>
  <files>src/lib/utils/url-params.ts</files>
  <action>
Refactor the `getUrlParams()` function to eliminate dynamic property injection:

1. Change return type from `Record<string, string>` to `Map<string, string>`
2. Use `Map.set(key, value)` instead of `params[key] = value`
3. Map has no prototype chain, eliminating prototype pollution entirely
4. Keep existing dangerous key filtering (**proto**, constructor, prototype) as defense-in-depth
5. Add JSDoc explaining the security rationale for Map usage

Example transformation:

```typescript
// Before (vulnerable to CodeQL warning)
const params: Record<string, string> = Object.create(null);
params[key] = value;

// After (Map has no prototype pollution)
const params = new Map<string, string>();
params.set(key, value);
```

Keep all other parsing functions (parseNumberParam, parseStringParam, parseBooleanParam) unchanged - they accept string | null and are not affected.

Update function signature:

```typescript
export function getUrlParams(): Map<string, string> {
  if (typeof window === "undefined") return new Map();

  const params = new Map<string, string>();
  const searchParams = new URLSearchParams(window.location.search);

  searchParams.forEach((value, key) => {
    // Defense in depth - filter dangerous keys even though Map is safe
    if (key !== "__proto__" && key !== "constructor" && key !== "prototype") {
      params.set(key, value);
    }
  });

  return params;
}
```
