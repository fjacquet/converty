  <files>src/lib/converters/infrastructure/vm-storage.ts</files>
  <action>
Create pure calculation function following VMware storage formulas from RESEARCH.md.

**Input Interface** (`VmStorageInput`):

- `vmConfigs`: Array of `{ diskGb: number; ramGb: number; count: number }` - VM profiles
- `includeSwapFiles`: boolean - Whether to allocate swap space
- `configLogGbPerVm`: number - Config/log overhead per VM (default: 0.25)
- `snapshotPercent`: number - Snapshot allocation percentage (0-100)
- `esxHosts`: number - Number of ESX hosts
- `esxStorageGbPerHost`: number - ESX installation overhead per host (default: 8)
- `thinProvisioningPercent`: number - Over-subscription percentage (0-100)
- `growthPercent`: number - Future growth allocation (0-100)

**Result Interface** (`VmStorageResult`):

- `totalProvisionedGb`: Total provisioned disk across all VMs
- `usedDiskGb`: Actual used disk (with thin provisioning)
- `overSubscribedGb`: Over-subscribed capacity (provisioned - used)
- `snapshotGb`: Snapshot allocation
- `swapGb`: Swap file allocation (sum of all VM RAM if enabled)
- `configLogGb`: Config/log files allocation
- `totalVmStorageGb`: Sum of used + snapshot + swap + configLog
- `esxStorageGb`: ESX hosts overhead
- `growthAllocationGb`: Growth headroom
- `totalRequiredGb`: Grand total required storage
- `totalVmCount`: Total number of VMs
- `percentages`: Object with percentage breakdown of each component
- `steps`: Array of calculation step descriptions

**Calculation formulas** (from RESEARCH.md):

1. `totalProvisionedGb = sum(diskGb * count)` for all VM configs
2. `usedDiskGb = totalProvisionedGb * (1 - thinProvisioningPercent / 100)`
3. `snapshotGb = totalProvisionedGb * (snapshotPercent / 100)`
4. `swapGb = includeSwapFiles ? sum(ramGb * count) : 0`
5. `configLogGb = totalVmCount * configLogGbPerVm`
6. `totalVmStorageGb = usedDiskGb + snapshotGb + swapGb + configLogGb`
7. `esxStorageGb = esxHosts * esxStorageGbPerHost`
8. `subtotal = totalVmStorageGb + esxStorageGb`
9. `growthAllocationGb = subtotal * (growthPercent / 100)`
10. `totalRequiredGb = subtotal + growthAllocationGb`

**Validation rules** (return null for invalid):

- `vmConfigs.length === 0` - No VM profiles
- Any negative values in vmConfigs
- Percentages outside 0-100 range
- `esxHosts < 1`

Include comprehensive JSDoc comments and `steps: string[]` for calculation transparency.
