# Milestone v5.0: Calculator Expansion

**Status:** ✅ SHIPPED 2026-01-29
**Phases:** 31-36
**Total Plans:** 6 phases (executed as single-plan phases with multi-wave delivery)

## Overview

v5.0 adds three new calculator domains to Converty:

1. **Engineering & CAD** - Professional engineering calculations (6 calculators)
2. **Chemistry & Science** - Laboratory and academic chemistry tools (6 calculators)
3. **Hyper-V & Virtualization Platform** - Multi-hypervisor infrastructure planning (6 calculators)

**Architectural Impact:** None - All calculators use existing v1.0-v4.0 patterns
**Bundle Impact:** ~90-110 KB (~18-22% increase)
**Risk Level:** MEDIUM (formula accuracy, reference data quality, licensing complexity)

## Milestone Summary

**Decimal Phases:** None (all phases executed in order)

**Key Decisions:**

- Custom recursive descent formula parser over npm dependency (zero-dependency chemistry parsing)
- Colebrook-White iterative solver with Newton-Raphson method (custom implementation, no external math library)
- NIST SP 811 12-digit precision for unit conversions (matches official standard)
- IUPAC 2024 atomic weights (latest standard)
- Platform-specific storage multipliers (thin × snapshot × replication stacking)
- Windows Server 16-core minimum enforcement in licensing calculator
- Backward-compatible multi-platform extension (default: VMware preserves v4.0 URLs)

**Issues Resolved:**

- TypeScript union type issues with platform-specific JSON data (biome-ignore with documentation)
- Hypervisor overhead data structure mismatches (nested objects vs flat numbers)
- Licensing data access path corrections (nested platform structure)
- Variable shadowing in power/labor calculations (renamed to avoid conflicts)
- Missing Alert UI component (replaced with styled Card)
- 476 missing translation keys across 4 locales (all resolved)
- 73 orphaned Italian translation keys cleaned up

**Issues Deferred:**

- Finite Element Analysis (FEA) — desktop software domain
- 3D molecular visualization — WebGL complexity
- Equation auto-balancing — high complexity, limited value
- Cloud hybrid comparison (on-prem vs AWS/Azure) — future enhancement

**Technical Debt Incurred:**

- 19 noArrayIndexKey warnings for static lists (acceptable, lists never reorder)
- biome-ignore annotations for necessary `any` casts in JSON platform data
- Licensing pricing requires manual quarterly updates (staleness warnings implemented)
