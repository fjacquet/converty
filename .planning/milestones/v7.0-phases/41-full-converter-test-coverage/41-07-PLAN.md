**Web converters:**

**csp.test.ts** — CSP string parsing/generation. Test: parse valid CSP header string → policy object, generate CSP from policy object → header string, roundtrip parse/generate is equivalent, null for empty input.

**emoji-chars.test.ts** — Unicode codepoint helpers. Test: known emoji character maps to expected codepoint, codepoint to emoji roundtrip, emoji name lookup returns expected character (check source for function names).

**html-chars.test.ts** — HTML entity lookup. Test: `&amp;` → `&`, `&lt;` → `<`, `&gt;` → `>`, unknown entity returns null or unchanged. Test both decode direction.

**html-encoder.test.ts** — Entity encoding/decoding. Test: encode `<script>alert('xss')</script>` → all special chars escaped, decode `&lt;script&gt;` → `<script>`, roundtrip encode/decode recovers original, empty string returns empty string.

**https-check.test.ts** — Pure URL parsing (NOT the async real HTTPS check). Test the `analyzeUrl()` or URL parsing function for: `http://example.com` → not HTTPS, `https://example.com` → is HTTPS, mixed content detection. Skip any functions making real network calls.

**redirect-check.test.ts** — `analyzeRedirectChain()` pure function. Test with constructed redirect chain objects (not real network). Test: empty chain returns no redirects, chain with http→https → upgrade detected, circular redirect detected. Read source for actual function signatures.

**seo-performance.test.ts** — SEO metrics scoring. Test: perfect title length (50-60 chars) → high title score, too short (<30) or too long (>70) → lower score, no H1 → lower headings score. null for empty metrics input.

**spf-check.test.ts** — SPF record string parser. Test: valid SPF `"v=spf1 include:example.com -all"` → parsed mechanisms array, invalid SPF format → error result, `include:` mechanism extracted correctly.

**url-encoder.test.ts** — encodeURIComponent wrapper. Test: `encodeUrl("hello world")` → `"hello%20world"`, special chars encoded, `decodeUrl("hello%20world")` → `"hello world"`, already-encoded input handled.

**Datetime converters:** All tests must use fixed date strings.

**age.test.ts** — Date diff in years/months/days. Test: `new Date("2000-01-15")` born, reference date `new Date("2024-01-15")` → age 24 years exactly, born `new Date("2000-06-15")` reference `new Date("2024-01-15")` → 23 years (before birthday), null for future birth date.

**date.test.ts** — Date formatting helpers. Test: format `new Date("2024-06-15")` as "YYYY-MM-DD" → "2024-06-15", ISO format, day of week. Check source for exported functions.

**day-counter.test.ts** — Business days, weekday counting. Test: weekdays between Mon "2024-01-01" and Fri "2024-01-05" = 5, same range excluding holidays if supported, null for end < start.

**day-of-week.test.ts** — Zeller's formula or Date API. Test: `new Date("2024-01-01")` → Monday (or numeric equivalent), `new Date("2024-06-15")` → Saturday. Verify day name matches expected.

**duration-converter.test.ts** — Hours/min/sec conversions. Test: 3661 seconds → 1h 1m 1s, 0 → 0h 0m 0s, negative → null or handled.

**hours.test.ts** — Decimal hours ↔ HH:MM. Test: 1.5 hours → "1:30", "2:45" → 2.75 hours, 0 → "0:00".

**time.test.ts** — Time arithmetic. Test: add 90 minutes to "10:00" → "11:30", subtract 45 minutes from "01:00" → "00:15", midnight wraparound.

**time-duration.test.ts** — Duration formatting. Test: 3600 seconds → "1 hour", 90 seconds → "1 minute 30 seconds", 0 → some representation.

**time-zone.test.ts** — Uses Intl.DateTimeFormat (Node.js 20 supports this). Test: calculateTimeZone function with hardcoded timezone IDs. Use "UTC" and "America/New_York" as test timezones. Test: getTimezoneOptions() returns an array with length > 0 (don't test exhaustively). calculateTimeZone for a fixed ISO datetime in UTC returns expected offset. DO NOT test Intl.supportedValuesOf exhaustively.
