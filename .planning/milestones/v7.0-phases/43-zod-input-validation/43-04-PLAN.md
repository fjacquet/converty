  <action>
Create the directory `src/lib/schemas/math/` and file `src/lib/schemas/math/index.ts`.

Read each math calculator component to determine its FormValues type, then write the matching Zod schema.

Math-specific patterns and constraints:

1. NUMERIC STRING FIELDS (most math calculators): `z.coerce.number()`
   - Percentages: `.min(0).max(100, "Percentage must be 0–100%")`
   - Angles: `.min(0).max(360, "Angle must be 0–360 degrees")` or radians range
   - Sample sizes: `.int().positive()`
   - Confidence level: `.min(0).max(1, "Confidence level must be 0–1")` or 0–100 depending on component

2. STRING INPUTS that represent formulas/lists (big-number, statistics, matrix):
   Use `z.string().min(1)` — don't try to validate content, just require non-empty

3. INTEGER-ONLY fields: `.int("Must be a whole number")`
   - n and r in permutation/combination: `.int().positive()`
   - Number of terms in sequence: `.int().positive()`

4. SPECIAL CASES:
   - binary-calculator: input is binary/hex string — use `z.string().min(1)` + `.regex(/^[01]+$/, "Must be binary (0s and 1s)")`
   - hex-calculator: hex string — use `z.string().regex(/^[0-9a-fA-F]+$/, "Must be hexadecimal")`
   - quadratic-calculator: a, b, c coefficients (can be any real number including negative) — `z.coerce.number()`
   - slope-calculator: x1, y1, x2, y2 — `z.coerce.number()` (any real number)
   - matrix-calculator: rows/columns input string — `z.string().min(1)` for text areas
   - big-number: text input — `z.string().min(1)`

Calculators to cover (read each component):

NOTE: basic-calculator in math may be a simple 4-function calculator — if its FormValues is trivial or it doesn't use InputField, use a minimal schema or skip wiring.

Naming convention: PascalCase + "FormSchema" — e.g., PercentageFormSchema, QuadraticFormSchema.

Apply the same three-step pattern to each math calculator component:

STEP 1 — Import schema:
```typescript
import { XxxFormSchema } from "@/lib/schemas/math";
```

STEP 2 — Add schema to store:
```typescript
const useStore = createCalculatorStore<FormValues, XxxResult | null>({
  schema: XxxFormSchema,
  // ... rest unchanged
});
```

STEP 3 — Destructure errors, pass to InputField:
```typescript
const { values, setValue, result, errors } = useStore();

<InputField
  id="value"
  ...
  error={errors.value}
/>
```

Math-specific rules:
- For string-input fields (textarea-style inputs in matrix, statistics, big-number), if the component uses a textarea or custom input instead of InputField, do NOT add error prop — just add the schema to the store for structural validation
- Quadratic: add error to a, b, c fields
- Percentage: add error to both numeric InputField components
- Sample size: add error to margin of error, confidence level, proportion fields

SKIP: `basic-calculator` if it has no InputField or is a simple button-based calculator.

IMPORTANT: If any calculator's FormValues needs a schema not in math/index.ts, add it there first.
