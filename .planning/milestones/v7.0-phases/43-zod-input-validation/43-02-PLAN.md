  <action>
Create the directory `src/lib/schemas/health/` and the file `src/lib/schemas/health/index.ts`.

Read each health calculator component to determine its FormValues type, then write a matching Zod schema.

PATTERN for string-based numeric fields (most health calculators):
```typescript
import { z } from "zod";
export const BmiFormSchema = z.object({
  weight: z.coerce.number({ error: "Weight must be a number" })
    .positive("Weight must be positive")
    .max(1000, "Weight exceeds maximum (1000)"),
  weightUnit: z.enum(["kg", "lb"]),
  height: z.coerce.number({ error: "Height must be a number" })
    .positive("Height must be positive")
    .max(300, "Height exceeds maximum (300 cm)"),
  heightUnit: z.enum(["cm", "m", "in", "ft"]),
});
```

Rules to follow for ALL schemas in this file:
- Use `z.coerce.number()` for fields where the FormValues type is `string` but represents a number
- Use `z.number()` for fields that are already numeric in the store's `initialValues`
- Use `z.enum([...])` for union string types (WeightUnit, HeightUnit, Sex, ActivityLevel, etc.)
- Use `z.string().min(1)` for free-form string fields (dates, formulas)
- Range constraints: apply physiologically sensible limits — e.g., age: 1–130, weight: 1–1000 kg, height: 1–300 cm
- Error messages in English (i18n deferred to Phase 46)

Export a named schema for EACH of the 28 health calculators. Inspect each calculator component to get its exact FormValues type. Calculator components to cover:

Naming convention: PascalCase + category + "FormSchema" — e.g., BmiFormSchema, BmrFormSchema, TdeeFormSchema.

```typescript
const useStore = createCalculatorStore<FormValues, XxxResult | null>({
  name: "xxx-calculator",
  initialValues: { ... },
  schema: XxxFormSchema,     // ADD THIS LINE
  calculate: (vals) => { ... },

```

CHANGE 3 — Destructure `errors` and pass to InputField:
```typescript
// Before: const { values, setValue, result } = useStore();
// After:
const { values, setValue, result, errors } = useStore();

// On EACH InputField that maps to a numeric or validated field:
<InputField
  id="weight"
  label={t("weightSimple")}
  value={values.weight}
  onChange={(v) => setValue("weight", v)}
  error={errors.weight}    // ADD THIS LINE
/>
```

IMPORTANT NOTES:
- Only add `error={errors.fieldName}` to InputField components — NOT to SelectField, RadioGroup, or custom UI elements
- The fieldName in errors.fieldName must match the key in FormValues exactly (e.g., errors.weight not errors.Weight)
- For enum fields (weightUnit, heightUnit, sex, activityLevel), do NOT add error prop — they are dropdowns with fixed valid options
- If a calculator component does NOT use InputField at all (custom UI), skip the errors wiring for that calculator but still add the schema param
- Ensure initialValues in the store still pass the schema (no validation errors on page load)

If any calculator's FormValues type does not have a matching schema in health/index.ts (missed in Task 1), add the missing schema to health/index.ts first before wiring.
