For EACH health converter file (excluding types.ts and index.ts):

1. Add import at top (after existing imports):
   ```typescript
   import type { CalculationResult } from "@/types";
   ```

2. Change function return type from `ResultType | null` to `CalculationResult<ResultType>`:
   ```typescript
   // Before
   export function calculateBMI(input: BMIInput): BMIResult | null {
   // After
   export function calculateBMI(input: BMIInput): CalculationResult<BMIResult> {
   ```

3. Replace each `return null` with a descriptive error return:
   ```typescript
   // Before
   if (input.weight <= 0 || input.height <= 0) {
     return null;
   }
   // After
   if (input.weight <= 0 || input.height <= 0) {
     return { ok: false, error: "Weight and height must be positive", code: "INVALID_INPUT" };
   }
   ```

4. Wrap the success return:
   ```typescript
   // Before
   return { bmi, category, healthyWeightRange, weightToHealthy };
   // After
   return { ok: true, value: { bmi, category, healthyWeightRange, weightToHealthy } };
   ```

5. For functions that never return null (always return a result), wrap the single return:
   ```typescript
   return { ok: true, value: result };
   ```

Error message guidance per null guard pattern:
- `<= 0` guard → `"INVALID_INPUT"` with message stating what must be positive
- Missing required string → `"INVALID_INPUT"` with message about required field
- Math domain error (sqrt of negative, log of zero) → `"CALCULATION_ERROR"`
- Division by zero → `"DIVISION_BY_ZERO"`

Each error message must be human-readable (lowercase, descriptive). Use past tense or must-form: "Height must be positive", "Weight must be greater than zero".

IMPORTANT: Update the corresponding store file IF a custom store exists in `src/stores/` for this converter (check file list — health stores that wrap custom calculate logic). The store's `calculate` option must now pass a function returning `CalculationResult<R>`.
