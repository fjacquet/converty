  <action>
Create the VM storage calculator with pure calculation functions following the established converter pattern.

**Input Interface (VmStorageInput):**
- vmCount: number (1-10000)
- diskSizeGb: number (1-10000, disk size per VM in GB)
- ramGb: number (1-1024, RAM per VM in GB)
- snapshotPercent: number (0-100, percentage of disk for snapshots)
- thinProvisioningPercent: number (0-100, over-subscription percentage for thin provisioning)
- provisioningType: "thick" | "thin"
- raidLevel: "raid0" | "raid1" | "raid5" | "raid6" | "raid10"

**Result Interface (VmStorageResult):**
- provisionedStorageGb: number (total VM disk storage)
- swapStorageGb: number (RAM-based swap files)
- snapshotStorageGb: number (snapshot space)
- totalRawGb: number (sum before RAID)
- raidOverheadGb: number (RAID overhead)
- totalRequiredGb: number (final total with RAID)
- totalRequiredTb: number (converted to TB)
- efficiencyPercent: number (usable / total)
- storagePerVmGb: number (average per VM)

**RAID Overhead Constants (RAID_LEVELS):**
- raid0: factor 1.0 (no redundancy)
- raid1: factor 2.0 (mirroring)
- raid5: factor 1.33 (single parity, assumes 3-disk minimum)
- raid6: factor 1.5 (double parity, assumes 4-disk minimum)
- raid10: factor 2.0 (mirror + stripe)

**Calculation Formulas:**
1. Provisioned Storage = vmCount * diskSizeGb
2. If thin: Actual Provisioned = Provisioned * (100 - thinProvisioningPercent) / 100
3. Swap Storage = vmCount * ramGb (VMware allocates swap = RAM)
4. Snapshot Storage = Provisioned * snapshotPercent / 100
5. Total Raw = Provisioned + Swap + Snapshot
6. RAID Overhead = Total Raw * (RAID_FACTOR - 1)
7. Total Required = Total Raw + RAID Overhead

**Implementation Notes:**
- Return null for invalid inputs (vmCount <= 0, diskSizeGb <= 0, etc.)
- Include exports for RAID_LEVELS and PROVISIONING_TYPES for UI dropdowns
- Include helper function formatStorageSize(gb) for display formatting
- Use typical VMware sizing assumptions (well-documented)
