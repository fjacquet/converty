CATEGORY-SPECIFIC NOTES:

**Photo (~22 calculators):**
- Most are numeric string fields: aperture, shutter speed, ISO, focal length
- Aperture: `z.coerce.number().positive().max(64, "f-stop too large")`
- ISO: `z.coerce.number().positive().max(1638400, "ISO too large")`
- Megapixels: `z.coerce.number().positive().max(1000)`
- DPI: `z.coerce.number().positive().max(10000)`

**Video (~9 calculators):**
- Bitrate: `z.coerce.number().positive()`
- Duration: `z.coerce.number().positive()`
- Frame rate: `z.coerce.number().positive().max(1000)`

**Data (~3 calculators + 2 converters):**
- File size values: `z.coerce.number().positive()`
- Speed: `z.coerce.number().positive()`
- Unit enums: `z.enum([...])` — check actual enum values used

**DateTime (~8 calculators):**
- Date string fields: `z.string().min(1)` (they are date picker inputs)
- Hour/minute values: `z.coerce.number().int().min(0).max(59)` for time fields
- Year values: `z.coerce.number().int().min(1900).max(2200)`

**Automotive (4 calculators):**
- Fuel efficiency: `z.coerce.number().positive()`
- Distance: `z.coerce.number().positive()`
- Tire dimensions: `z.coerce.number().positive().int()`
- Vehicle price: `z.coerce.number().positive()`
- Interest rate: `z.coerce.number().min(0).max(100)`
- Currency enum: `z.enum(["CHF", "EUR"])` (from Phase 41 decision: Automotive Currency is CHF|EUR only)

**Cooking (4 calculators):**
- Serving size: `z.coerce.number().positive()`
- Ingredient amounts: `z.coerce.number().positive()`
- Food cost: `z.coerce.number().positive()`

**Network (~5 calculators):**
- IP address fields: `z.string().min(7).max(45)` (IPv4 or IPv6)
- CIDR prefix: `z.coerce.number().int().min(0).max(128)`
- Bandwidth: `z.coerce.number().positive()`
- Latency: `z.coerce.number().positive()`
- SKIP subnet-calculator custom UI for error wiring (still add schema to store)

**Crypto (4 calculators):**
- Hash rate: `z.coerce.number().positive()`
- Exchange rate: `z.coerce.number().positive()`
- Wallet address: `z.string().min(1)` (free-form, validated by custom logic)

**Web (~10 calculators):**
- URL string: `z.string().min(1)` (or z.string().url() for URL validators)
- CSP/SPF text: `z.string().min(1)`
- Numeric SEO scores: `z.coerce.number().int().min(0).max(100)`

**Physics (1 calculator — speed converter):**
- Speed value: `z.coerce.number().positive()`
- Unit enum: `z.enum([...])` — check actual speed units

**Music (1 calculator — BPM):**
- BPM value: `z.coerce.number().positive().max(300, "BPM too large")`

**Color (1 converter — RGB):**
- R, G, B values: `z.coerce.number().int().min(0).max(255, "Color value must be 0–255")`
- Or hex string: `z.string().regex(/^#?[0-9a-fA-F]{6}$/, "Invalid hex color")`
