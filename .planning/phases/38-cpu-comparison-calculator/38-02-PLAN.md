**Component structure:**

1. **Filter panel** — Card with title from `t("filterPanel")`:
   - Vendor dropdown: options `all`, `intel`, `amd`, `arm` — labels from `t("vendors.all")`, `t("vendors.intel")`, `t("vendors.amd")`, `t("vendors.arm")`; on change call `setValue("vendor", value)` and reset `cpuIds` to `""` to avoid stale selections
   - Generation dropdown: options `all`, `current`, `previous` — labels from `t("generations.all")`, `t("generations.current")`, `t("generations.previous")`; on change call `setValue("generation", value)` and reset `cpuIds` to `""`

2. **CPU selection panel** — Card with title from `t("selectCpus")` and subtitle hint `t("selectCpusHint")` (e.g., "Select 2 to 4 CPUs to compare"):
   - Derive `filteredCpus` using `useMemo(() => getFilteredCpus(values.vendor, values.generation), [values.vendor, values.generation])`
   - Parse selected IDs: `const selectedIds = values.cpuIds ? values.cpuIds.split(",").filter(Boolean) : []`
   - Render a checkbox list (or multi-select checkboxes) for each CPU in `filteredCpus`; checkbox checked if `selectedIds.includes(cpu.id)`
   - On checkbox change: add or remove the ID from `selectedIds`, clamp to max 4, join with comma, call `setValue("cpuIds", newValue)`
   - Disable unchecked checkboxes when `selectedIds.length >= 4` (max reached)
   - Show selected count badge: `t("selectedCount", { count: selectedIds.length })`

3. **Results table** — only render when `result !== null` (i.e., 2+ CPUs selected):
   - **Staleness warning**: if `result.isStale`, show an `AlertCircle` warning card with `result.staleWarning` and a link to `https://www.spec.org/cpu2017/results/cint2017.html`
   - **Table layout** (responsive): vertical table where rows = metrics, columns = CPUs (max 4 columns)
   - Table rows (all labels from translations):
     | Row label | Value |
     |-----------|-------|
     | `t("cpuName")` | `row.name` |
     | `t("vendor")` | `row.vendor` (capitalize first letter) |
     | `t("family")` | `row.family` |
     | `t("generation")` | `t("generations." + row.generation)` |
     | `t("cores")` | `row.cores` |
     | `t("tdpW")` | `row.tdpW + " W"` |
     | `t("socketType")` | `row.socketType` |
     | `t("specint2017Base")` | `row.specint2017Base` |
     | `t("specint2017Peak")` | `row.specint2017Peak` |
     | `t("perfPerCore")` | `row.perfPerCore` |
     | `t("perfPerWatt")` | `row.perfPerWatt` |
     | `t("sizingRatio")` | `row.sizingRatioVsFirst === 1.0 ? t("baseline") : row.sizingRatioVsFirst + "x"` |
   - Highlight the baseline (first) CPU column with a subtle background distinction (e.g., `bg-muted/40` on header cell)
   - Use Tailwind `overflow-x-auto` on table wrapper for mobile horizontal scroll
   - Use `<table>` HTML element with `<thead>`, `<tbody>`, `<th>`, `<td>` for semantic markup

4. **Empty state**: When `result === null` and `selectedIds.length < 2`, show a muted hint card: `t("emptyState")` (e.g., "Select at least 2 CPUs to see the comparison")
