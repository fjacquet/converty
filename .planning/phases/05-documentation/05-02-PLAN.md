**Title:** Use Zustand for Calculator State Management

**Metadata:**
- Status: accepted
- Date: 2026-01-15 (Phase 3 completion date)
- Deciders: Project team

**Context and Problem Statement:**
Calculators needed shareable URLs and predictable state management. Previous implementation used useState with manual URL sync, leading to code duplication across 117 calculators. How should we manage state?

**Decision Drivers:**
- Need URL state synchronization for shareable calculator links
- Want to eliminate boilerplate in calculator components
- Must maintain type safety with TypeScript strict mode
- Prefer minimal bundle size impact
- Must support 117+ calculators without performance issues

**Considered Options:**
1. Zustand with custom middleware
2. Redux Toolkit with RTK Query
3. Jotai with URL persistence
4. Continue with useState pattern and refactor URL sync

**Decision Outcome:**
Chosen option: "Zustand with custom middleware"

Positive consequences:
- Minimal bundle size (1.2KB gzipped)
- Custom createCalculatorStore factory eliminates boilerplate
- URL sync middleware provides shareable links automatically
- No Provider wrapper needed (simpler architecture)
- Type-safe with generics

Negative consequences:
- Learning curve for contributors unfamiliar with Zustand
- Required migration of 117 existing calculators
- Less ecosystem tooling compared to Redux

- createCalculatorStore implementation: src/stores/calculator-store.ts
- Example usage: src/app/[locale]/datetime/age/age-calculator.tsx
