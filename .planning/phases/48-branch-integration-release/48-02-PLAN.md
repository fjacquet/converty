    ## v7.0 Framework Migration

    This milestone adopts proven building blocks — Vitest, Zod, react-error-boundary, Sonner, DOMPurify, LZ-String, discriminated union result types, and a restructured i18n schema — without replacing Next.js.

    **169 calculators. 4 locales. All still working.**

    ## What Changed

    ### Phase 40: Vitest Foundation
    - Vitest configured in Next.js project (node environment, v8 coverage)
    - 66 tests across 5 priority converters with ≥75% per-file coverage

    - ADR-011 documents the test strategy

    ### Phase 41: Full Converter Test Coverage
    - **2288+ tests** across 196 test files covering all 19 converter categories
    - 86% line coverage, 91% branch coverage on \`src/lib/converters/\`
    - Global 75% coverage threshold enforced in \`vitest.config.ts\`
    - CI gate added to \`static.yml\` — tests run before every build

    ### Phase 42: Error Boundaries & Toast Notifications
    - \`react-error-boundary\` wraps all 169 calculators in \`ConverterLayout\`
    - Sonner toast notifications for copy-to-clipboard, CSV/PDF export
    - \`isomorphic-dompurify\` sanitize utility for safe HTML rendering
    - Opt-in \`onCalculationError\` callback in \`createCalculatorStore\`

    ### Phase 43: Zod v4 Input Validation
    - Zod v4.3.6 as a runtime dependency
    - 15 category schema files in \`src/lib/schemas/\` — 100+ input schemas
    - Optional \`schema\` parameter added to \`createCalculatorStore\`
    - Zod URL param helpers (\`zod-url-params.ts\`) for type-safe URL parsing
    - Field-level validation errors wired into 71 store-based components

    ### Phase 44: LZ-String URL Compression
    - \`lz-string@1.5.0\` for URL state compression (60-80% length reduction)
    - Compressed state written as \`?z=\` search param (not hash — GitHub Pages compatible)
    - Backward-compatible: legacy per-key URLs continue to work
    - Prototype pollution prevention: only \`initialValues\` keys accepted from parsed JSON
    - Round-trip lossless compression verified in tests

    ### Phase 45: Discriminated Union Result Types
    - \`CalculationResult<T>\`: \`{ ok: true; value: T } | { ok: false; error: string; code: string }\`
    - Adapter pattern: stores stay backward compatible, discriminant unwrapped internally
    - \`calculationError: string | undefined\` added to \`CalculatorState\`
    - 91 store-based components display typed error messages beneath inputs
    - All 196 test files updated to use \`.ok\` discriminant assertions

    ### Phase 46: i18n Namespace Restructure
    - All 4 locale JSON files restructured to stable 4-key schema: \`common\`, \`nav\`, \`converter\`, \`calculator\`
    - ~210 source files updated: \`converters\` → \`converter\`, \`categories\` → \`nav\`
    - Zero \`MISSING_MESSAGE\` warnings in production build
    - ADR-012 documents the rationale and migration path

    ### Phase 47: ADRs & CI Hardening
    - ADR-011: Vitest test strategy
    - ADR-013: LZ-String URL compression (search params rationale)
    - ADR-014: Zod validation layer architecture
    - ADR-015: Discriminated union result types (adapter pattern)
    - CI \`static.yml\` now includes type-check gate alongside test gate

    ### Phase 48: Branch Integration & Release
    - Full regression suite: type-check + tests + build + Biome — all green
    - CHANGELOG.md updated with complete v7.0 entries
