## Setup

1. Build production static export:
   ```bash
   npm run build
   ```

2. Serve build locally (install serve if needed):
   ```bash
   npx serve out
   ```
   Note the local URL (typically http://localhost:3000)

3. Open in Chrome/Edge for full PWA testing (Safari has limited PWA support)

## Verification Steps

### PWA-01: Service Worker for Offline Functionality

**Test in Chrome DevTools:**

1. Open browser DevTools (F12)
2. Go to Application tab -> Service Workers section
3. **Expected:** Service worker registered and Status shows "activated and is running"
4. Note the Scope: should be "/" (root)

**Test offline caching:**

1. Visit a calculator page (e.g., /en/finance/mortgage or any calculator)
2. Let page fully load
3. Open DevTools -> Application tab -> Cache Storage
4. **Expected:** See caches: "workbox-precache-v2-..." and runtime caches (pages-cache, static-cache)
5. Click on precache, **Expected:** See HTML, JS, CSS files listed
6. Go to Network tab -> Enable "Offline" checkbox (simulates airplane mode)
7. Refresh the page (Ctrl+R / Cmd+R)
8. **Expected:** Page still loads and calculator works (served from cache, no network errors)
9. Try using calculator - change inputs, see results
10. **Expected:** All calculations work offline (client-side JavaScript)

**Success criteria:**
- ✓ Service worker shows "activated" status
- ✓ Cache Storage contains precached assets
- ✓ Page loads while offline (Network tab shows "from ServiceWorker")
- ✓ Calculator functionality preserved offline

### PWA-02: Web Manifest for Install Prompt

**Test manifest:**

1. DevTools -> Application tab -> Manifest section
2. **Expected:** See manifest loaded with:
   - Name: "Converty - Calculators & Converters"
   - Short name: "Converty"
   - Start URL: "/"
   - Display: "standalone"
   - Icons: 4 icons listed (192x192, 512x512, maskable, apple-touch-icon)
3. Click icon URLs to verify images load

**Test install prompt (Desktop Chrome/Edge):**

1. Look for install icon in address bar (right side, near bookmark star)
2. **Expected:** Install icon appears (if installability criteria met)
3. Click install icon
4. **Expected:** Dialog appears: "Install Converty?"
5. Click "Install"
6. **Expected:** App opens in standalone window (no browser UI)
7. Close app, check Start menu / Applications
8. **Expected:** Converty appears as installed app

**Alternative: Install button in page (if implemented in layout):**

1. If InstallPrompt component added to layout, look for "Install Converty" button
2. Click button
3. **Expected:** Browser install dialog appears

**Success criteria:**
- ✓ Manifest loads in DevTools with correct values
- ✓ All 4 icons load successfully
- ✓ Install prompt available (address bar icon or custom button)
- ✓ App installs successfully and launches standalone

### PWA-03: Mobile-Optimized Responsive Design

**Test responsive design:**

1. DevTools -> Toggle device toolbar (Ctrl+Shift+M / Cmd+Shift+M)
2. Select mobile device: iPhone 12 Pro or Pixel 5
3. Navigate to several calculators
4. **Expected:**
   - Layouts adapt to narrow screen (no horizontal scroll)
   - Input fields touch-friendly (minimum 44px tap targets)
   - Text readable without zooming
   - Calculator grid/tables responsive

**Test on real mobile device (if available):**

1. Deploy to test URL or use ngrok to expose local server
2. Open on iPhone or Android phone
3. Navigate through calculators
4. **Expected:** Same responsive behavior as DevTools testing

**Success criteria:**
- ✓ All calculators usable on 375px viewport
- ✓ No horizontal scrolling required
- ✓ Touch targets appropriately sized
- ✓ Content readable without zoom

### PWA-04: Offline Fallback UI

**Test offline banner:**

1. Online mode: Visit any calculator page
2. **Expected:** No offline banner visible (isOnline = true)
3. DevTools Network tab -> Enable "Offline" checkbox
4. **Expected:** Yellow banner appears at top: "You are offline. Calculators will work from cache."
5. Verify banner is visible and not hidden by other UI
6. Network tab -> Uncheck "Offline" (back online)
7. **Expected:** Banner disappears immediately (online/offline events working)

**Test in actual offline scenario:**

1. Disconnect from WiFi or use airplane mode on device
2. Navigate to calculator
3. **Expected:** Offline banner appears
4. Try using calculator
5. **Expected:** Calculator still works, banner remains visible
6. Reconnect to network
7. **Expected:** Banner disappears

**Success criteria:**
- ✓ Offline banner appears when network disconnected
- ✓ Banner disappears when network reconnected
- ✓ Banner provides clear message about offline status
- ✓ Banner doesn't interfere with calculator usability
