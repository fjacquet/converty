   - 192x192px (Chrome minimum)
   - 512x512px (Chrome minimum)
   - 192x192px maskable variant (Android adaptive icon with 80% safe zone)
   - 180x180px apple-touch-icon (iOS home screen)

Install Sharp if needed: `npm install -D sharp`

Run the script to generate all icon files.

Create app/manifest.ts using Next.js built-in manifest support (NOT static manifest.json - research shows app/manifest.ts is preferred for type safety).

Implement manifest function returning MetadataRoute.Manifest with:

- name: "Converty - Calculators & Converters"
- short_name: "Converty"
- description: "200+ free online calculators and converters for finance, health, math, photo, video, and more"
- start_url: "/" (root, Next.js handles locale routing)
- display: "standalone" (app-like experience)
- background_color: "#ffffff" (matches site theme)
- theme_color: "#000000" (matches header/dark theme)
- icons array with:
  - icon-192x192.png (sizes: "192x192", type: "image/png", purpose: "any")
  - icon-512x512.png (sizes: "512x512", type: "image/png", purpose: "any")
  - icon-192-maskable.png (sizes: "192x192", type: "image/png", purpose: "maskable")
  - apple-touch-icon.png (sizes: "180x180", type: "image/png")

Use Next.js MetadataRoute.Manifest type for type safety.

WHY app/manifest.ts: Research shows Next.js 16 App Router provides built-in manifest support with type checking. Static manifest.json works but lacks TypeScript validation and dynamic generation capabilities.

IMPORTANT: Do NOT add locale prefix to icon paths. Icons are static assets served from public/ root, not locale-specific routes.
