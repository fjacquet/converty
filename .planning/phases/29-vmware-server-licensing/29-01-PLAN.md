  <files>src/lib/converters/infrastructure/server-virtualization.ts</files>
  <action>
Create pure calculation function for ESX host sizing following VMware best practices.

**Input Interface** (`ServerVirtualizationInput`):

- `vmCount`: number - Total number of VMs to host
- `vCpuPerVm`: number - vCPU per VM (e.g., 2, 4, 8)
- `ramPerVmGb`: number - RAM per VM in GB (e.g., 8, 16, 32)
- `hostCores`: number - Physical CPU cores per ESX host (e.g., 32, 48)
- `hostRamGb`: number - Total RAM per ESX host in GB (e.g., 256, 512)
- `vCpuToCoreRatio`: number - vCPU-to-core over-subscription ratio (default: 4:1)
- `targetCpuUtilization`: number - Target CPU utilization percentage (0-100, default: 80)
- `targetRamUtilization`: number - Target RAM utilization percentage (0-100, default: 85)
- `highAvailability`: boolean - Enable N+1 HA (adds one extra host)

**Result Interface** (`ServerVirtualizationResult`):

- `totalVCpuRequired`: number - Total vCPU across all VMs
- `totalRamRequiredGb`: number - Total RAM across all VMs
- `effectiveCpuPerHost`: number - Host cores × vCpuToCoreRatio × (targetCpuUtilization/100)
- `effectiveRamPerHostGb`: number - Host RAM × (targetRamUtilization/100)
- `hostsNeededByCpu`: number - Hosts needed to meet CPU requirements (ceiling)
- `hostsNeededByRam`: number - Hosts needed to meet RAM requirements (ceiling)
- `hostsNeededBeforeHa`: number - Max of CPU/RAM constraints
- `hostsNeededTotal`: number - Final host count (with HA if enabled)
- `limitingFactor`: "cpu" | "ram" - Which resource constrains capacity
- `vCpuConsolidationRatio`: number - Actual vCPU per physical core
- `finalCpuUtilization`: number - Actual CPU % after rounding hosts
- `finalRamUtilization`: number - Actual RAM % after rounding hosts
- `steps`: string[] - Calculation step descriptions

**Calculation formulas**:

1. `totalVCpuRequired = vmCount × vCpuPerVm`
2. `totalRamRequiredGb = vmCount × ramPerVmGb`
3. `effectiveCpuPerHost = hostCores × vCpuToCoreRatio × (targetCpuUtilization / 100)`
4. `effectiveRamPerHostGb = hostRamGb × (targetRamUtilization / 100)`
5. `hostsNeededByCpu = Math.ceil(totalVCpuRequired / effectiveCpuPerHost)`
6. `hostsNeededByRam = Math.ceil(totalRamRequiredGb / effectiveRamPerHostGb)`
7. `hostsNeededBeforeHa = Math.max(hostsNeededByCpu, hostsNeededByRam)`
8. `limitingFactor = hostsNeededByCpu > hostsNeededByRam ? "cpu" : "ram"`
9. `hostsNeededTotal = highAvailability ? hostsNeededBeforeHa + 1 : hostsNeededBeforeHa`
10. `vCpuConsolidationRatio = totalVCpuRequired / (hostsNeededTotal × hostCores)`
11. `finalCpuUtilization = (totalVCpuRequired / (hostsNeededTotal × hostCores × vCpuToCoreRatio)) × 100`
12. `finalRamUtilization = (totalRamRequiredGb / (hostsNeededTotal × hostRamGb)) × 100`

**Validation**:

- Return null if vmCount, vCpuPerVm, ramPerVmGb, hostCores, or hostRamGb ≤ 0
- Return null if vCpuToCoreRatio ≤ 0 (must be positive)
- Return null if targetCpuUtilization or targetRamUtilization not in 1-100 range

**Include JSDoc comments** explaining formulas and N+1 HA pattern.
**Add `steps` array** with 8-10 calculation step descriptions for transparency.

<task type="auto">
  <name>Task 2: Create VMware licensing calculation logic</name>
  <files>src/lib/converters/infrastructure/vmware-licensing.ts</files>
  <action>
Create pure calculation function for VMware VCF/VVF licensing costs with core-based pricing.

**Input Interface** (`VmwareLicensingInput`):

- `hostCount`: number - Number of ESX hosts
- `cpusPerHost`: number - Number of CPU sockets per host (typically 2)
- `coresPerCpu`: number - Cores per CPU socket (e.g., 16, 24, 32)
- `productType`: "vcf" | "vvf" | "vsphere-ep" | "vsphere-std"
- `termYears`: 1 | 3 | 5 - Subscription term

**Pricing constants** (2026 list prices):

```typescript
const PRICING = {
  vcf: 350,        // VMware Cloud Foundation - $350/core/year
  vvf: 135,        // VMware vSphere Foundation - $135/core/year
  "vsphere-ep": 95,   // vSphere Enterprise Plus - $95/core/year (estimated)
  "vsphere-std": 50,  // vSphere Standard - $50/core/year (estimated)
} as const;

const VSAN_ENTITLEMENT = {
  vcf: 1.0,    // 1 TiB per core
  vvf: 0.25,   // 0.25 TiB per core
} as const;
```

**Result Interface** (`VmwareLicensingResult`):

- `totalPhysicalCores`: number - Total cores across all hosts (cpus × cores × hosts)
- `totalLicensedCores`: number - Cores after 16-core minimum enforcement
- `coresPerCpuLicensed`: number - Cores licensed per CPU (max of actual or 16)
- `pricePerCorePerYear`: number - Annual price per core
- `annualCost`: number - Total annual cost
- `totalCost`: number - Total cost over term
- `vsanEntitlementTib`: number | null - vSAN storage entitlement (null for vSphere-only)
- `minCoreEnforced`: boolean - True if 16-core minimum was applied
- `steps`: string[] - Calculation step descriptions

**Calculation formulas**:

1. `totalPhysicalCores = hostCount × cpusPerHost × coresPerCpu`
2. `coresPerCpuLicensed = Math.max(coresPerCpu, 16)` - VMware 16-core minimum per CPU
3. `totalLicensedCores = hostCount × cpusPerHost × coresPerCpuLicensed`
4. `pricePerCorePerYear = PRICING[productType]`
5. `annualCost = totalLicensedCores × pricePerCorePerYear`
6. `totalCost = annualCost × termYears`
7. `vsanEntitlementTib = productType in VSAN_ENTITLEMENT ? totalLicensedCores × VSAN_ENTITLEMENT[productType] : null`
8. `minCoreEnforced = coresPerCpu < 16`

**Validation**:

- Return null if hostCount, cpusPerHost, or coresPerCpu ≤ 0
- Return null if productType not in valid list
- Return null if termYears not in [1, 3, 5]

**Include JSDoc comments** explaining VMware licensing model (core-based, 16-core minimum, vSAN entitlement).
**Add `steps` array** with 6-8 calculation step descriptions.
