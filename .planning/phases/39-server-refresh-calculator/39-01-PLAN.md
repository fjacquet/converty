`ServerRefreshInput` (URL-syncable, all fields strings for createCalculatorStore):
- `oldCpuId: string` — CPU ID from the database for the old fleet
- `oldSocketsPerServer: string` — sockets per old server (e.g. "2")

- `chassisConstraint: string` — "none" | "1u-single" | "2u-dual"

`FleetSummary`:
- `serverCount: number`
- `socketsPerServer: number`
- `totalSockets: number`
- `totalCores: number`
- `totalTdpW: number`
- `totalSpecint: number` — total SPECint2017 peak (sockets × CPU specint2017Peak)

`ServerRefreshResult`:
- `oldFleet: FleetSummary`
- `newFleet: FleetSummary`
- `requiredSpecint: number` — oldFleet.totalSpecint × (1 + headroomPct/100)
- `minNewServerCount: number` — Math.ceil(requiredSpecint / specintPerNewServer)
- `specintPerNewServer: number` — newCpu.specint2017Peak × socketsPerNewServer
- `headroomPct: number`
- `powerBudgetW: number` — 0 if none
- `serversPerRack: number | null` — Math.floor(powerBudgetW / newFleet single server TDP) or null
- `racksNeeded: number | null` — Math.ceil(newFleet.serverCount / serversPerRack) or null
- `dataAsOf: string`
- `isStale: boolean`
- `staleWarning?: string`

**Function: `calculateServerRefresh(input: ServerRefreshInput): ServerRefreshResult | null`**

Logic:
1. Parse all string inputs to numbers; return null if oldServerCount <= 0 or oldSocketsPerServer <= 0 or newSocketsPerServer <= 0
2. Look up oldCpu and newCpu from cpuDatabase.cpus by id; return null if either not found
3. Apply chassis constraint to effective socketsPerNewServer:
   - "1u-single" → socketsPerNewServer = 1
   - "2u-dual" → socketsPerNewServer = Math.min(parsed value, 2)
   - "none" → use parsed value as-is
4. Compute oldFleet summary:
   - totalSockets = oldSocketsPerServer × oldServerCount
   - totalCores = totalSockets × oldCpu.cores
   - totalTdpW = oldServerCount × (oldSocketsPerServer × oldCpu.tdpW)
   - totalSpecint = totalSockets × oldCpu.specint2017Peak
5. Compute requiredSpecint = oldFleet.totalSpecint × (1 + headroomPct / 100)
6. Compute specintPerNewServer = effectiveSockets × newCpu.specint2017Peak
7. Compute minNewServerCount = Math.ceil(requiredSpecint / specintPerNewServer)
8. Build newFleet summary using minNewServerCount and effectiveSockets
9. Power budget: if powerBudgetW > 0, compute singleServerTdpW = effectiveSockets × newCpu.tdpW; serversPerRack = Math.floor(powerBudgetW / singleServerTdpW); racksNeeded = Math.ceil(minNewServerCount / serversPerRack). If powerBudgetW <= 0, both null.
10. Staleness: same pattern as cpu-comparison.ts (daysOld > staleDays → isStale)
11. Return full result; return null for any division-by-zero edge case (specintPerNewServer === 0)

Round all computed numbers: Math.round for TDP totals and specint totals, Math.ceil for server/rack counts.

**Function: `getServerRefreshCpus(): CpuEntry[]`**
Returns all CPUs from database sorted by specint2017Peak descending. Used for both old and new CPU selectors.

Note: Do NOT re-export `getFilteredCpus` from cpu-comparison — use the dedicated `getServerRefreshCpus()` to avoid naming conflicts. The server refresh calculator uses a single combined list without vendor/generation filter (user picks specific models).
