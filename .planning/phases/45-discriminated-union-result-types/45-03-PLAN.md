**SPECIAL CASE: recipe-scaler.ts and food-cost.ts — Convert throws to CalculationResult:**

These two files currently throw errors for invalid inputs (confirmed in Phase 41 tests). Convert to return-based error handling:

```typescript
// Before (throw pattern)
export function calculateRecipeScaler(input: RecipeInput): RecipeResult {
  if (input.servings <= 0) {
    throw new Error("Servings must be positive");
  }
  // ...
  return result;
}

// After (CalculationResult pattern)
import type { CalculationResult } from "@/types";

export function calculateRecipeScaler(input: RecipeInput): CalculationResult<RecipeResult> {
  if (input.servings <= 0) {
    return { ok: false, error: "Servings must be positive", code: "INVALID_INPUT" };
  }
  // ...
  return { ok: true, value: result };
}
```

Read recipe-scaler.ts and food-cost.ts first to identify ALL throw sites, then convert each throw to a `{ ok: false, error: originalMessage, code: "INVALID_INPUT" }` return.
