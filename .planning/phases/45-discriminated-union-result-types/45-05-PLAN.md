**SPECIAL CASE 1: network/supernetting.ts — success field rename**

Read supernetting.ts first. It uses `success: boolean` inline in its return type (not a pure discriminated union). Migration steps:
1. Remove the existing return type annotation (or change it to `CalculationResult<SupernetPayload>`)
2. Change all `success: true` → `ok: true, value: { ...payload }` (extracting the non-success fields into value)
3. Change all `success: false` → `ok: false, error: "...", code: "INVALID_INPUT"`
4. Add `code` field to all error returns
5. Update test: `result.success` → `result.ok`, `result.networks` → `result.value.networks` etc.

**SPECIAL CASE 2: chemistry/formula-parser.ts — ParseResult type rename**

Read formula-parser.ts first. It uses `ParseResult = { success: true; formula: ... } | { success: false; error: string }`. Migration steps:
1. Replace `ParseResult` type entirely with `CalculationResult<FormulaPayload>`
2. Change `success: true` → `ok: true, value: { formula, elements, ... }` (all success fields into value)
3. Change `success: false, error: "..."` → `ok: false, error: "...", code: "INVALID_INPUT"`
4. Update callers within the file if any
5. Update test: `result.success` → `result.ok`, `result.formula` → `result.value.formula`

**SPECIAL CASE 3: chemistry/equation-parser.ts — same pattern as formula-parser.ts**

**Engineering converters** (excluding types.ts, index.ts): beam-deflection, column-buckling, moment-of-inertia, pipe-flow, stress-strain, unit-converter.
Standard migration: return null → { ok: false }, success return → { ok: true, value: ... }

**Web converters** (excluding index.ts): csp, emoji-chars, html-chars, html-encoder, https-check, redirect-check, seo-performance, spf-check, url-encoder.

IMPORTANT for web converters: Some web converters (spf-check, url-encoder) have `success` as a FIELD INSIDE their result object, not as the return type discriminant. These are different:
- If function returns `WebResult | null` and WebResult contains `{ success: boolean, ... }` → wrap entire WebResult as value: `return { ok: true, value: webResult }`. The `value.success` field stays unchanged inside the result.
- Only rename `success` → `ok` at the RETURN TYPE LEVEL (the discriminant of the return type itself).
- Do NOT rename internal `success` fields inside the result payload.

Read each web converter before migrating to determine which pattern applies.
