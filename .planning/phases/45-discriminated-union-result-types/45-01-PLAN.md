```typescript
/**
 * Discriminated union result type for all calculator functions.
 * Replaces `T | null` to carry typed error information on failure.
 *
 * @example
 * // Success path
 * return { ok: true, value: result };
 *
 * // Failure path
 * return { ok: false, error: "Weight must be positive", code: "INVALID_INPUT" };
 */
export type CalculationResult<T> =
  | { ok: true; value: T }
  | { ok: false; error: string; code: string };
```

Recommended code values (non-exhaustive, use as appropriate):
- `"INVALID_INPUT"` — missing or out-of-range input
- `"DIVISION_BY_ZERO"` — mathematical singularity
- `"CALCULATION_ERROR"` — unexpected calculation failure
- `"UNSUPPORTED_INPUT"` — input type not supported

Then update `src/types/index.ts` to re-export:
```typescript
export * from "./converter";
export * from "./calculation-result";
```

No enum needed — TypeScript string type is sufficient for code field.
