<task type="auto">
  <name>Task 2: Create build-time search index generator</name>
  <files>
    scripts/generate-search-index.ts
  </files>
  <action>
Create `scripts/generate-search-index.ts` that generates locale-specific search indexes:

1. Import converters from registry and translations from all 4 locale files
2. For each locale (en, fr, de, it):
   - Map each converter to a SearchDocument with translated name/description
   - Use `messages.converters[converter.id]?.name` for translated name
   - Use `messages.categories[converter.category]?.name` for translated category
   - Build href as `/${converter.category}/${converter.slug}`
3. Create Fuse.js index with keys: name, description, keywords, categoryName
4. Serialize both documents and index to JSON
5. Write to `public/search/index-{locale}.json`

Key implementation details:
- Use `node:fs` and `node:path` for file operations
- Use Fuse.createIndex() to pre-build the search index
- Export index via `myIndex.toJSON()` for serialization
- Create public/search/ directory if it doesn't exist

Fuse options for index creation:
```typescript
const keys = ['name', 'description', 'keywords', 'categoryName'];
const index = Fuse.createIndex(keys, documents);
```

Output format per file:
```json
{
  "documents": [...SearchDocument[]],
  "index": {...FuseIndex}
}
```

Create `src/lib/search/search-index.ts` with lazy-loading search initialization:

```typescript
const fuseOptions: IFuseOptions<SearchDocument> = {
  keys: [
    { name: 'name', weight: 0.4 },
    { name: 'description', weight: 0.2 },
    { name: 'keywords', weight: 0.3 },
    { name: 'categoryName', weight: 0.1 },
  ],

  includeScore: true,
  ignoreLocation: true,  // Don't penalize matches at end of string

};
```
