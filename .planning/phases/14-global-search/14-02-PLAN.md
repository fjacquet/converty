  <action>
Create `src/components/search/global-search.tsx` as a client component:

1. State management:
   - `open: boolean` - dialog open state, initialize as false
   - `query: string` - search input
   - `results: SearchDocument[]` - search results
   - `fuse: Fuse<SearchDocument> | null` - cached Fuse instance
   - `isLoading: boolean` - loading state for index fetch

2. Keyboard shortcut effect (Cmd+K / Ctrl+K):
```typescript
useEffect(() => {
  const down = (e: KeyboardEvent) => {
    if (e.key === "k" && (e.metaKey || e.ctrlKey)) {
      e.preventDefault();
      setOpen((prev) => !prev);
    }
  };
  document.addEventListener("keydown", down);
  return () => document.removeEventListener("keydown", down);
}, []);
```

3. Load search index when dialog opens (lazy loading):
```typescript
useEffect(() => {
  if (open && !fuse) {
    setIsLoading(true);
    getSearchInstance(locale).then((instance) => {
      setFuse(instance);
      setIsLoading(false);
    });
  }
}, [open, locale, fuse]);
```

4. Search with React's useDeferredValue for smooth typing:
```typescript
const deferredQuery = useDeferredValue(query);

useEffect(() => {
  if (fuse && deferredQuery) {
    setResults(search(deferredQuery, fuse));
  } else {
    setResults([]);
  }
}, [deferredQuery, fuse]);
```

5. Handle result selection:
```typescript
const handleSelect = useCallback((doc: SearchDocument) => {
  router.push(`/${locale}${doc.href}`);
  setOpen(false);
  setQuery("");
}, [router, locale]);
```

6. Render structure:
   - Search trigger button with Search icon and "Search..." text
   - Show keyboard shortcut hint (Cmd+K on Mac, Ctrl+K on Windows)
   - Dialog with Command inside (NOT CommandDialog - use Dialog + Command separately)
   - Set `shouldFilter={false}` on Command since Fuse.js handles filtering
   - CommandInput for search input
   - CommandList with CommandEmpty for no results
   - CommandGroup for results, CommandItem for each result
   - Show calculator name and category in results

Important: Use Dialog from `@/components/ui/dialog`, NOT cmdk's built-in dialog, for consistent styling.
