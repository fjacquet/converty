Create `src/lib/middleware/url-sync.ts` implementing Zustand middleware factory with per-instance debounce timer isolation.

Implementation requirements:

1. **Middleware Factory Signature:**

   - Export `createUrlSyncMiddleware<T extends object>(options?: UrlSyncOptions)`
   - Returns middleware function with signature: `(config: StateCreator<T, [], []>): StateCreator<T, [], []>`
   - CRITICAL: Timer declared in factory scope (not module scope) creates closure

2. **UrlSyncOptions Interface:**

   ```typescript
   interface UrlSyncOptions {
     enabled?: boolean; // default: true
     debounceMs?: number; // default: 150
     include?: string[]; // only sync these keys (default: all)
     exclude?: string[]; // exclude these keys (default: none)
   }
   ```

3. **Closure Pattern (STATE-04 fix):**

   ```typescript
   export function createUrlSyncMiddleware<T extends object>(
     options: UrlSyncOptions = {}
   ) {
     // CLOSURE: Timer declared HERE creates NEW closure per factory call
     let debounceTimeout: ReturnType<typeof setTimeout> | null = null;

     // Return middleware function...
   }
   ```

4. **Middleware Logic:**

   - Load initial state from URL parameters on store creation
   - Wrap `api.setState` to add debounced URL sync after each update
   - Clear previous timeout before scheduling new one (debounce pattern)
   - Use `window.history.replaceState` (NOT pushState - avoids history pollution)
   - Guard all browser APIs with `typeof window === "undefined"` (SSG compatibility)

5. **URL Sync Logic:**

   - Build URLSearchParams from state object
   - Filter by include/exclude options
   - Skip functions, undefined, null, empty string values
   - Atomic update: build complete search params before single replaceState call
   - Only sync values matching include list (if specified)
   - Skip values in exclude list

6. **TypeScript Requirements:**

   - Import `StateCreator` from zustand
   - Preserve generic type `T` through middleware chain
   - Use proper StateCreator signature: `StateCreator<T, [], []>`
   - Export all types (UrlSyncOptions, createUrlSyncMiddleware)

7. **Reference Implementation:**
   Use research example from `02-RESEARCH.md` section "Complete URL Sync Middleware Implementation" as primary reference. This is the proven pattern for Zustand middleware with closure-captured state.

**IMPORTANT: Why NOT to use global module-level timer:**
The current `calculator-store.ts` has `let debounceTimeout` at module scope (line 48). This is shared across ALL store instances. When Store A updates, it clears Store B's pending timeout, causing lost updates. Factory-scoped timer creates NEW closure per invocation, ensuring isolation.

5. **Middleware Integration Pattern:**

   ```typescript
   export function createCalculatorStore<T extends object, R>({
     initialValues,
     calculate,
     validate,
     syncUrl = true,
     debounceMs = 150,
   }: CreateCalculatorStoreOptions<T, R>) {
     const storeCreator: StateCreator<CalculatorState<T, R>> = (set, get) => ({
       // State and methods WITHOUT inline URL sync
       values: initialValues,
       result: null,
       errors: {},
       setValue: /* ... */,
       setValues: /* ... */,
       reset: /* ... */
     });

     // Apply middleware conditionally
     if (syncUrl) {
       return create<CalculatorState<T, R>>()(
         createUrlSyncMiddleware<CalculatorState<T, R>>({
           debounceMs,
           include: Object.keys(initialValues), // Only sync 'values' fields
         })(storeCreator)
       );
     }

     return create<CalculatorState<T, R>>()(storeCreator);
   }
   ```
