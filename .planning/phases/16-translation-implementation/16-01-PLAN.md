<task type="auto">
  <name>Task 1: Fix i18n Check Script False Positives</name>
  <files>scripts/check-i18n.js</files>
  <action>
Fix the false positive detection in the i18n check script. The script incorrectly reports `calculator.finance.currencies` and `calculator.finance.states` as missing keys because it treats namespace declarations as key usage.

The bug is in the `extractKeysFromSource` function around lines 100-143. When the `keyPattern` regex matches `useTranslations("calculator.finance.currencies")`:
- It captures `varName = "useTranslations"` and `key = "calculator.finance.currencies"`
- Since "useTranslations" is not in the namespaces Map, it falls through to the else branch
- Since the key contains a dot, it's added to the keys Set - incorrectly!

Fix by excluding `useTranslations` and `getTranslations` from the key pattern matching. In the keyPattern loop (around line 102), add a check to skip when varName is one of the translation function names:

```javascript
// Skip if this is a translation function declaration, not a key usage
if (varName === "useTranslations" || varName === "getTranslations") {
  match = keyPattern.exec(content);
  continue;
}
```

Add this check immediately after line 105 (`const key = match[2];`) and before the numeric check on line 109.
  </action>
  <verify>
Run `node scripts/check-i18n.js` and confirm it reports "All translation keys are present!" with zero missing keys for all 4 locales.
  </verify>
  <done>
The i18n check script no longer reports false positives for namespace patterns like `useTranslations("calculator.finance.currencies")`.
  </done>
