  <files>src/lib/converters/infrastructure/k8s-capacity.ts</files>
  <action>
Create pure calculation function following Kubernetes capacity formulas from RESEARCH.md.

**Input Interface** (`K8sCapacityInput`):

- `podCpuRequest`: number - CPU millicores per pod (e.g., 1000m = 1 core)
- `podMemoryRequest`: number - Memory MiB per pod
- `podReplicas`: number - Number of pod instances
- `daemonSetCpuPerNode`: number - DaemonSet CPU overhead per node (millicores)
- `daemonSetMemoryPerNode`: number - DaemonSet memory overhead per node (MiB)
- `nodeCpuCores`: number - Total CPU cores per node
- `nodeMemoryMb`: number - Total memory per node in MB
- `systemReservedCpu`: number - System reserved CPU (millicores, OS + K8s)
- `systemReservedMemory`: number - System reserved memory (MiB, OS + K8s + eviction)
- `targetCpuUtilization`: number - Target CPU utilization percentage (0-100)
- `targetMemoryUtilization`: number - Target memory utilization percentage (0-100)

**Result Interface** (`K8sCapacityResult`):

- `allocatableCpuPerNode`: number - Available CPU per node (millicores)
- `allocatableMemoryPerNode`: number - Available memory per node (MiB)
- `totalPodCpuRequired`: number - Total CPU for all pods (millicores)
- `totalPodMemoryRequired`: number - Total memory for all pods (MiB)
- `targetCpuPerNode`: number - Target CPU accounting for utilization (millicores)
- `targetMemoryPerNode`: number - Target memory accounting for utilization (MiB)
- `nodesNeededByCpu`: number - Nodes needed based on CPU constraint
- `nodesNeededByMemory`: number - Nodes needed based on memory constraint
- `nodesNeededTotal`: number - Max of CPU/memory (limiting factor)
- `limitingFactor`: "cpu" | "memory" - Which resource constrains capacity
- `finalCpuUtilization`: number - Actual CPU % after rounding nodes
- `finalMemoryUtilization`: number - Actual memory % after rounding nodes
- `overUtilized`: boolean - True if either resource exceeds 80%
- `steps`: string[] - Calculation step descriptions

**Calculation formulas** (from RESEARCH.md Pattern 3):

1. **Allocatable resources per node**:
   - `allocatableCpuPerNode = nodeCpuCores * 1000 - systemReservedCpu - daemonSetCpuPerNode`
   - `allocatableMemoryPerNode = nodeMemoryMb - systemReservedMemory - daemonSetMemoryPerNode`

2. **Total pod requirements**:
   - `totalPodCpuRequired = podCpuRequest * podReplicas`
   - `totalPodMemoryRequired = podMemoryRequest * podReplicas`

3. **Target allocatable (accounting for utilization)**:
   - `targetCpuPerNode = allocatableCpuPerNode * (targetCpuUtilization / 100)`
   - `targetMemoryPerNode = allocatableMemoryPerNode * (targetMemoryUtilization / 100)`

4. **Nodes needed by each constraint**:
   - `nodesNeededByCpu = Math.ceil(totalPodCpuRequired / targetCpuPerNode)`
   - `nodesNeededByMemory = Math.ceil(totalPodMemoryRequired / targetMemoryPerNode)`

5. **Select limiting factor (max of CPU/memory)**:
   - `nodesNeededTotal = Math.max(nodesNeededByCpu, nodesNeededByMemory)`
   - `limitingFactor = nodesNeededByCpu > nodesNeededByMemory ? "cpu" : "memory"`

6. **Calculate final utilization with rounded node count**:
   - `finalCpuUtilization = (totalPodCpuRequired / (allocatableCpuPerNode * nodesNeededTotal)) * 100`
   - `finalMemoryUtilization = (totalPodMemoryRequired / (allocatableMemoryPerNode * nodesNeededTotal)) * 100`

7. **Check for over-utilization warning**:
   - `overUtilized = finalCpuUtilization > 80 || finalMemoryUtilization > 80`

**Validation rules** (return null for invalid):

- `podReplicas < 1` - No pods
- `nodeCpuCores < 1` - Invalid node
- `nodeMemoryMb < 1` - Invalid node
- `podCpuRequest <= 0` - Invalid pod request
- `podMemoryRequest <= 0` - Invalid pod request
- `targetCpuUtilization < 1 || targetCpuUtilization > 100` - Invalid percentage
- `targetMemoryUtilization < 1 || targetMemoryUtilization > 100` - Invalid percentage
- `systemReservedCpu < 0` - Negative overhead
- `systemReservedMemory < 0` - Negative overhead
- `daemonSetCpuPerNode < 0` - Negative overhead
- `daemonSetMemoryPerNode < 0` - Negative overhead
- `allocatableCpu <= 0` - System overhead exceeds node capacity
- `allocatableMemory <= 0` - System overhead exceeds node capacity

Include comprehensive JSDoc comments and populate `steps: string[]` array with calculation breakdown (see RESEARCH.md Pattern 3 for step format).
